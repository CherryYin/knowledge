# Knowledgebase API Guide

## Run

- Start server: `uvicorn main:app --reload --port 8000`
- Health check: `GET /healthz`

## Environment Variables

### Ingestion Backend

- File/Text ingestion uses built-in local pipeline (`doc_parser` + `ingestion_pipeline`).
- No Azure Search or Azure Document Intelligence config is required for file/text ingestion.

### TLS / Certificates (recommended for corporate networks)

If you see `SSLCertVerificationError` when calling Azure AD (`login.microsoftonline.com`), configure one of:

- `SSL_CA_BUNDLE=/absolute/path/to/your-ca.pem` (preferred; can be a company root CA bundle)
- `SSL_VERIFY=false` (temporary workaround; disables certificate verification)

### Embeddings

- `EMBEDDING_BASE_URL` (vLLM service URL, default: `http://localhost:8100`)
- `EMBEDDING_MODEL` (model name, default: `Qwen/Qwen2.5-VL-Embedding`)
- `USE_LOCAL_EMBEDDING=true` (optional; switch to local Transformers model instead of vLLM)

### Document Parsing Strategy

- File ingestion uses the built-in local parser pipeline (`doc_parser` + `ingestion_pipeline`), without Azure Document Intelligence.
- `USE_LOCAL_PDF_PARSER` is no longer required for this path.

### FAQ Storage (required for FAQ ingestion)

- `FAQ_DB_PATH` (optional, default: `./data/faq.db`)

### External Segment Sync (optional)

When enabled, each chunk generated by file/text ingestion is POSTed to an external endpoint:

- `DOCUMENT_SEGMENTS_BASE_URL` (e.g. `http://127.0.0.1:8001`)
- `DOCUMENT_SEGMENTS_PATH` (optional, default: `/api/v1/document_segments`)
- `DOCUMENT_SEGMENTS_TOKEN` (Bearer token value, without `Bearer ` prefix)
- `DOCUMENT_SEGMENTS_TIMEOUT_SECONDS` (optional, default: `8`)

Payload fields per chunk:

- `source_type` (`file` or `text`)
- `document_id` (uses current `doc_id`)
- `content` (chunk text)
- `chunk_number` (0-based index)
- `start_page`, `end_page`
- `keywords` (section title/path fallback)

## APIs

Base URL examples below assume `http://localhost:8000`.

### 1) File Ingestion

Router: `/file-ingestion`

#### POST `/file-ingestion/upload`

Upload one or more files, parse via local pipeline, generate embeddings, and upsert into vector store.

- Content-Type: `multipart/form-data`
- Form field: `files` (repeatable)
- Required header: `X-User-Id`

Example:

```bash
curl -X POST "http://localhost:8000/file-ingestion/upload" \
  -H "X-User-Id: user-001" \
  -F "files=@/path/to/doc1.pdf" \
  -F "files=@/path/to/doc2.txt"
```

Response:

```json
{
  "count": 2,
  "results": [
    {"filename":"doc1.pdf","doc_id":"<sha256>","status":"success"},
    {"filename":"doc2.txt","doc_id":"<sha256>","status":"duplicate"}
  ]
}
```

#### DELETE `/file-ingestion/delete`

Body:

```json
{ "doc_id": "<sha256-doc-id>" }
```

Supported extensions (built-in parser): `.pdf`, `.docx`, `.pptx`, `.xlsx`, `.xls`, `.csv`, `.md`, `.markdown`, `.html`, `.htm`, `.txt`, `.text`.

### 2) FAQ Ingestion

Router: `/faq-ingestion`

Stores FAQ data in a persistent SQLite DB via `kblib/vector_db.py`.

#### POST `/faq-ingestion/upsert`

Body:

```json
{
  "id": 123,
  "question": "How do I reset my password?",
  "answer": "Go to settings -> security -> reset.",
  "markets": "" 
}
```

Example:

```bash
curl -X POST "http://localhost:8000/faq-ingestion/upsert" \
  -H "Content-Type: application/json" \
  -d '{"id":123,"question":"Q?","answer":"A","markets":""}'
```

Response:

```json
{ "id": 123 }

#### PUT `/faq-ingestion/update`

Update by id.

#### DELETE `/faq-ingestion/{faq_id}`

Deletes one FAQ vector by id.
```

#### POST `/faq-ingestion/reindex`

Clears the FAQ table then upserts all items.

Body:

```json
{
  "items": [
    {"id": 1, "question": "Q1", "answer": "A1", "markets": ""},
    {"id": 2, "question": "Q2", "answer": "A2", "markets": "CN"}
  ]
}
```

Response:

```json
{ "count": 2 }
```

### 3) Text Ingestion

Router: `/text-ingestion`

Vectorizes raw text and upserts into local vector store.

- Required header: `X-User-Id`

#### POST `/text-ingestion/upsert`

Body:

```json
{
  "text": "Your long text here...",
  "sourcefile": "my-text-001",
  "category": "general",
  "url": null
}
```

Example:

```bash
curl -X POST "http://localhost:8000/text-ingestion/upsert" \
  -H "X-User-Id: user-001" \
  -H "Content-Type: application/json" \
  -d '{"text":"hello world","sourcefile":"demo","category":"general"}'
```

Response:

```json
{ "status": "success", "doc_id": "<sha256>", "total_chunks": 1 }
```

If same `X-User-Id + sourcefile` already exists, API returns:

```json
{ "status": "duplicate", "doc_id": "<sha256>", "total_chunks": 0 }
```

#### PUT `/text-ingestion/update`

Delete old by deterministic `doc_id` then re-ingest.

#### DELETE `/text-ingestion/{doc_id}`

Deletes all indexed chunks by `doc_id`.

Notes:
- `doc_id = sha256(user_id + "|" + file_name)`.
- `file_name` uses original value (`filename` or `sourcefile`) as-is.
- If the token count is very large, the text will be split into multiple chunks before indexing.
